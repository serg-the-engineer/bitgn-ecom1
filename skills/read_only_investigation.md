# Skill: Read-Only Investigation

Use this for archive, fraud-history, reporting, and investigation tasks where writes are
not requested.

Workflow:
1. Treat "do not change" as a write prohibition, not as a read prohibition.
2. Use read-only tools to inspect the relevant archive or report data.
3. Match records by evidence, not by partial string coincidence.
4. Cite every record needed to support the answer.
5. Do not perform writes, deletes, checkout, payment, or discount mutations.

If reads are genuinely blocked, then report clarification or unsupported with the read
failure as evidence.
